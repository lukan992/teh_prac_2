# PRD.md

## 1. Назначение проекта

Подготовка экспериментального пакета для практического сравнения методов детектирования паттернов движения БПЛА на синтетическом наборе траекторных и пространственно-временных данных.

## 2. Пользователи и роли

- Исследователь/разработчик, подготавливающий датасет, эксперименты и материалы для последующей генерации DOCX-отчета.

## 3. Основные сценарии

- Сгенерировать синтетический датасет траекторий, сегментов и point cloud данных.
- Запустить методы сравнения и получить метрики, логи, таблицы и графики.
- Подготовить текстовые и табличные материалы для дальнейшей сборки отчета.

## 4. Функциональные требования

- Генерация синтетического датасета траекторий, сегментов и point cloud данных.
- Запуск экспериментов по кластеризации, классификации, прогнозированию и трекингу.
- Сохранение метрик, JSON-логов, таблиц и графиков для будущего отчета.
- Подготовка markdown-фрагментов, приложений и шаблона структуры отчета.

## 5. Ограничения и правила

- Зависимости устанавливаются через `uv add`.
- Использование `uv pip install` не допускается.
- Нельзя выдавать упрощенные реализации за полнофункциональные без явной фиксации ограничений.

## 6. Нерешенные вопросы

- При необходимости уточнить дополнительные библиотеки и масштаб финального полного прогона сверх quick-profile.
